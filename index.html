Gestion de usuarios

Un usuario es cualquier persona que usa un ordenador. Los nombres de usuario describen los nombres que representan a dichos usuarios. Esos nombres pueden ser cualquiera. Lo único que importa es que el equipo tenga un nombre para cada cuenta, y es con este nombre a través del cual una persona adquiere los permisos para usar el sistema. No obstante, algunos servicios del sistema se ejecutan utilizando cuentas de usuario restringidas o privilegiadas. Cualquier persona puede tener más de una cuenta, con un nombre diferente para cada cuenta que cree. Además, hay algunos nombres reservados, que no pueden ser utilizados, como «root». 

								
								drwxrwx--- 1 root vboxsf 16384 Jan 29 11:02 sf_Shared

En este ejemplo, la carpeta sf_Shared es propiedad del usuario root y del grupo vboxsf . También es posible determinar los propietarios y los permisos de un archivo con la orden stat:

Los permisos de acceso se muestran en tres grupos de caracteres, representando los permisos del usuario propietario, del grupo propietario, y de los otros, respectivamente. Por ejemplo, los caracteres -rw-r--r-- indican que el usuario propietario del archivo tiene permisos de lectura y escritura, pero no de ejecución (rw-), mientras que los usuarios que pertenecen al grupo propietario y los demás usuarios solo tienen permiso de lectura (r-- y r--).

Para visualizar una lista de los usuarios actualmente conectados al sistema, se puede usar la orden who


Para agregar un nuevo usuario, utilice la orden useradd:

								
										# useradd -m -g [grupo_principal] -G [grupos_adicionales] -s [shell_de_ingreso] [nombre_de_usuario]
								
							
										
												# useradd -m -g [grupo_principal] -G [grupos_adicionales] -s [shell_de_ingreso] [nombre_de_usuario]
  • -m crea el directorio home del usuario con la ruta /home/[nombre de usuario]
  • -g define el nombre o el número del grupo principal del inicio de sesión del usuario, el nombre del grupo debe existir
  • -G introduce una lista de grupos suplementarios de los que el usuario será miembro
  • -s define la ruta y el nombre de la shell predeterminada de inicio de sesión del usuario
											
													# useradd -m -g users -s /bin/bash archie

Gestion de grupos

  • Los grupos solo pueden contener usuarios, nunca a otros grupos
$ groups [usuario]

La orden groups muestra la pertenencia al grupo


$ cat /etc/group

Puede listar todos los grupos presentes en el sistema, ejecutando


# groupadd [grupo]

Puede crear un nuevo grupo con la orden


# gpasswd -a [usuario] [grupo]

Puede agregar un usuario a un grupo con la orden


# groupdel [grupo]

Puede eliminar un grupo existente, ejecutando


# gpasswd -d [usuario] [grupo]
